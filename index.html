프로덕트매니저(PM)를 목표로 공부 중

소개 역량 프로젝트 연락

사용자의 문제를 정의하고, 가설을 세우고, 데이터와 커뮤니케이션으로 제품을 개선하는 과정을 좋아합니다. 작은 실험을 빠르게 반복하며 “왜?”를 끝까지 파고드는 PM이 되고 싶습니다.

About

  • 관심 분야: 제품 전략, 사용자 리서치, MVP/실험 설계, 지표 기반 개선
  • 일하는 방식: 문제 정의 → 가설/우선순위 → 실행 → 측정 → 학습/반복
  • 목표: 사용자의 “불편”을 “경험”으로 바꾸는 프로덕트를 만드는 PM

지금 공부하는 것

PRD 작성 / 요구사항 정리
사용자 인터뷰 / 설문 설계
지표 설계 (North Star, Funnel)
A/B 테스트 기초
애자일 / 스프린트 운영
기본 HTML/CSS/JS

프로젝트 (예시 영역)

  • PM 스터디 노트 정리 페이지 — 학습 내용을 주차별로 정리하고 공유 (Notion/블로그 등)
  • 가상의 기능 개선 PRD — 문제 정의, KPI, 시나리오, 릴리즈 계획까지 작성
  • 간단한 랜딩 페이지 — HTML/CSS로 자기소개 1페이지 제작 (현재 페이지)

위 항목들은 예시입니다. 실제 경험/링크에 맞게 수정해 주세요.

핵심 역량

  • 문제 정의 및 우선순위 설정
  • 사용자 관점의 요구사항 정리
  • 팀 커뮤니케이션 및 조율
  • 지표 기반 의사결정(기초)
